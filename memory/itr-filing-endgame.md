Hard-won from filing AY2026-27 ITR-2 on a real AY2026-27 ITR-2 filing.

**The portal will NOT accept a hand-built/hand-edited JSON uploaded directly.** It enforces two server-side integrity controls, in this order:
1. Whitelisted Software Provider ID — "Invalid Software Provider ID ... does not have access to upload this Return."
2. Content-integrity Digest/hash — "Invalid hash value identified, Modification to ITR details outside Utility is not allowed."

So the **final upload file must be generated by the official ITD offline utility** (or a registered ERI). The utility computes the hash and stamps a valid provider ID on Save. **Do NOT forge the hash or spoof the provider ID** — they are integrity controls; declined to reverse-engineer them and that was correct.

**Correct workflow:** agent generates a schema-valid JSON → user imports it into the official utility → utility stamps hash+ID on Save/Generate → user uploads that file → e-verify. The utility's *import* does not need a valid hash (only upload does), so importing our generated JSON works.

**Numbers import as 0 unless the JSON carries ITEMIZED sub-arrays** (the utility rebuilds totals bottom-up):
- Salary: `ScheduleS.Salaries[].Salarys.NatureOfSalary.OthersIncDtls[]` ({NatureDesc:"1"=Basic, OthAmount:<17(1) gross>}).
- Schedule 112A: per-scrip `Schedule112ADtls[]` (ISINCode, ShareUnitName, sale/cost/balance, ShareOnOrBefore AE/BE) — reconciled to summary. CG summary page shows only NET gain; gross sale sits inside the 112A grid.
- OS and FA import fine from their scalar/array structure.
- Diagnostic when a schedule shows 0 but text imports: diff our JSON against the real portal prefill JSON (ITD-produced), which always ships both itemization and roll-up.

**Aadhaar (s.139AA):** `PartA_GEN1.PersonalInfo.AadhaarCardNo` (`[0-9]{12}`) is schema-optional but the portal blocks e-verify without it. Sourceable from the prefill (base64-encoded, not masked).

**Machine gotcha:** the ARM-only Mac desktop utility won't run on Intel Macs. Use Windows desktop utility, Apple-Silicon Mac utility, or the Excel .xlsm (MS Excel + macros; NOT LibreOffice). Confirm the user's machine EARLY.
